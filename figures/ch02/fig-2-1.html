Conway's Law vs Inverse Conway Maneuver
Conway's Law (thụ động)
Cấu trúc team hiện tại
Team A
Frontend + DB
Team B
Backend + API
↓ phản ánh
Module A
UI + Data
Module B
Logic + API
Kiến trúc phần mềm phản ánh cấu trúc team
Coupling chéo giữa modules
Kiến trúc = bản sao cấu trúc tổ chức
(không chủ động)
Inverse Conway (chủ động)
Kiến trúc phần mềm mong muốn
Service X
Service Y
↓ thiết kế team
Team X
Own Service X
Team Y
Own Service Y
Tái cấu trúc team phù hợp kiến trúc
Deploy độc lập theo service
Thiết kế team để đạt kiến trúc mong muốn
(chủ động)
Conway's Law: tổ chức tác động thụ động lên kiến trúc — Inverse Conway: chủ động thiết kế tổ chức phù hợp kiến trúc đích
