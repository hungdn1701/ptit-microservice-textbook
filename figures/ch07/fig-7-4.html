Các chiến lược Cache trong Microservices
1. Cache-Aside
(Lazy Loading)
App đọc Cache
↓
hit ↙
Cache HIT
→ Trả về ngay
miss ↘
Cache MISS
↓
Đọc DB
↓
Ghi vào Cache
→ Trả về
2. Read-Through
(Cache tự đọc DB)
App đọc Cache
↓ miss
Cache tự đọc DB
↓
Cache lưu + trả về
App chỉ làm việc với Cache
Cache tự quản lý DB-sync
Transparent to app layer
3. Write-Behind
(Async Write)
App ghi Cache
↓
Cache ghi DB async
↓ async / batch
Batch / Delayed write to DB
Ghi nhanh → trả về ngay
DB sync sau (eventual)
⚠ Có thể mất data nếu crash
Cache-Aside
App tự load khi cache miss. Dễ implement, phổ biến nhất (Redis + Spring Cache).
Read-Through
Cache layer tự đọc DB. App không biết DB tồn tại. Cần cache hỗ trợ.
Write-Behind
Ghi cache trước, sync DB sau async. Throughput cao nhưng rủi ro mất data.
