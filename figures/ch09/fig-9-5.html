JWT Validation — Gateway vs Services
Auth Service — Full Validation
1. Verify JWT signature (secret key)
2. Check expiry (exp claim)
3. Query DB: user exists? active?
Nặng, nhưng chính xác nhất
Core/Judge — Claims-Only
1. Read X-User-Id header (tu Gateway)
2. Read X-User-Roles header
3. Authorize based on roles
Nhẹ, nhanh, tin tưởng Gateway
Trade-off:
Gateway làm 'heavy lifting' (verify JWT). Services chỉ đọc headers = nhanh, ít coupling với Auth.
Rủi ro: Nếu bypass Gateway (internal call), không có validation. => Cần mutual TLS hoặc service mesh.
